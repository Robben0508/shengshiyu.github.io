Git小记

发表于 2018-03-16 | 分类于 Technology
| 字数统计 1,528 字



常用命令

设置和配置

  • git config

创建和获取项目

  • git init
    初始化本地git仓库（创建新仓库）

  • git clone
    clone远程仓库

快照

  • git add

  • git commit

  • git diff

  • git rm

  • git mv

  • git status
    用来查看当前版本状态，查看是否进行了修改

分支与合并

  • git log

  • git tag

  • git branch
    可以列出你所拥有的分支、创建的新分支、删除分支以及重命名分支

  • git checkout
    用于切换分支并将内容检出到工作目录

  • git merge
    用于将一个或多个分支合并，然后将合并的分支作为当前分支

  • git stash

项目共享和更新

  • git remote
    可以用来管理远程仓库记录，将一段很冗杂的URL地址保存成一个简单的字符串句柄，比如origin，方便操作

  • git push
    与其他仓库通信，确定本地内容和远程仓库异同，然后将差异推送到其他仓库

  • git fetch

  • git pull
    可以理解为git fetch和git merge的组合，先从远程仓库获取内容，再尝试合并进你所在分支

  • git reset

检视和比较

  • git show
    能够以简单易读的形式显示Git对象，通常用来显示标签或提交的相关信息

  • git ls-files

  • git show-branch

调试

  • git grep
    可以帮助你在源代码的所有文件中找到任意字符串

打补丁

  • git cherry-pick

  • git rebase
    基本上就是一个自动化的git cherry-pick命令，它确定一系列提交，然后在别处以相同的顺序逐个对其挑拣。

  • git revert
    与git cherry-pick命令的效果恰好相反，将变更撤销或还原

管理

  • git reflog
    显示所有提交，包括孤立节点
  • git gc
    执行”垃圾回收“，删除无用文件，将余下的文件打包成一种更有效的形式
  • git fsck
    用于检查内部数据库存在的问题或不一致性
